# Задание 3

## Анализ результатов

### 4 процесса
- Стабильно высокая эффективность (99-111%) для всех размеров
- Линейное ускорение ~4x достигается уже для матриц 512×512
- Минимальные накладные расходы - блоки достаточно крупные

### 16 процессов
Сильная зависимость от размера задачи:
- **256×256**: крайне низкая эффективность 3% (параллельная версия медленнее последовательной!)
- **512×512**: эффективность 24% - начинает окупаться
- **1024×1024**: хорошая эффективность 83%
- **2048×2048**: отличная эффективность 121% - суперлинейное ускорение

## Выводы

- **Алгоритм Кэннона** эффективен для крупномасштабных матричных умножений.
- При правильном выборе размера задачи относительно числа процессов достигается высокая эффективность и даже суперлинейное ускорение.
- Ключевой фактор успеха - соотношение объема вычислений к объему коммуникаций, которое должно быть достаточно большим для минимизации накладных расходов.
