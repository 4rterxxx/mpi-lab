# Задание 2

## Анализ результатов

### Лучший алгоритм: Разбиение по строкам
- Стабильно показывает наилучшие результаты для всех размеров матриц
- Максимальное ускорение: 1.86x при N=5040 на 4 процессах
- Наивысшая эффективность: 46% при N=5040 на 4 процессах

### Худший алгоритм: Блочное разбиение
- Наименьшая производительность в большинстве тестов
- Сильная деградация при увеличении числа процессов

### Маленькая матрица (N=576)
- Все алгоритмы неэффективны при p>4
- Причина: Накладные расходы MPI превышают выигрыш от распараллеливания
- Лучший результат: разбиение по строкам (ускорение 1.10x на 4 процессах)

### Средняя матрица (N=2016)
- Разбиение по строкам показывает умеренную эффективность (34%)
- Разбиение по столбцам и блокам значительно отстают
- Резкое падение производительности при 9+ процессах

### Большая матрица (N=5040)
- Разбиение по строкам демонстрирует хорошую масштабируемость до 4 процессов
- Все алгоритмы деградируют при 16 процессах

## Выводы

- **Разбиение по строкам** является наиболее эффективным методом для параллельного умножения матрицы на вектор. Оно обеспечивает лучшую локальность данных и минимальные коммуникационные расходы.
- **Блочное разбиение** показало наихудшие результаты из-за высоких накладных расходов на организацию коммуникаций между процессами.
- Эффективное распараллеливание достигается только для достаточно больших матриц (N > 2000) и при умеренном количестве процессов (2-4).
- Чрезмерное увеличение числа процессов приводит к доминированию коммуникационных расходов над вычислительными выгодами.
